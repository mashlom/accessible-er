# מה שלומי? · מסע נגיש במלר״ד ילדים

אפליקציית דמו להנגשת הביקור במיון ילדים (מלר״ד) עבור ילדים אוטיסטים ומשפחותיהם.
הכלי מלווה את ההורה והילד/ה לאורך המסע, מפחית אי־ודאות, ומכין לפרוצדורות —
בקצב שלהם ובשפה פשוטה.

> הכלי אינו מחליף את הצוות הרפואי ואינו מספק ייעוץ רפואי. זהו דמו לצורכי אפיון.

## מה יש בפנים

- **חשוב לדעת על הילד/ה שלי (כרטיס התאמות)** — ההורה מסמן במהירות (צ'יפים, בלי
  חובת שם) תקשורת, רגישויות, מה מרגיע, מה מסלים, קושי בבדיקות וביטוי כאב — ומקבל
  כרטיס קצר וגדול להצגה לצוות מהמסך או בהדפסה. סימון רגישות פותח הצעות להתאמות
  שכדאי לבקש. נשמר במכשיר בלבד עם תפוגה של 24 שעות; שום דבר לא נשלח.
- **למה הגענו?** — בחירת סיבת ההגעה (חום, כאבי בטן, חבלה, חתך, אחר). הבחירה
  נשמרת ל־24 שעות (`localStorage` עם תפוגה), כך שרענון דף שומר עליה, ואפשר לעדכן
  בכל רגע. לפי הבחירה מוצג המסלול הצפוי.
- **מה קורה עכשיו?** — שלבי הביקור כ"סטפר" (כולל ייעוץ מומחה), עם משמעות לכל שלב,
  הערכות זמן בטווחים, מה אפשר לבקש, וקישורי הכנה לפרוצדורות של אותו שלב.
  אפשר לסמן "אנחנו כאן" (נשמר ליום).
- **מה עומד לקרות?** — ספריית הכנה חזותית ל־16 פרוצדורות (כולל בדיקת רופא/ה
  והמתנה לתוצאות), עם "סיפור" מודרך שלב־אחר־שלב עם דמות מלווה (רוני).
- **איפה אנחנו?** — מפת התמצאות עם שכבת "עומס חושי" (שקט / בינוני / עמוס)
  ו"לאן ממשיכים מכאן" לכל אזור.
- **מה עושים כשקשה?** — הנחיות קצרות להורה + משפטים מוכנים לצוות שאפשר להציג
  בגדול על המסך ברגע שקשה לדבר.
- **מה אפשר לבקש?** — התאמות לגיטימיות שאפשר לבקש מהצוות.
- **כניסה מהירה ב־QR** — דף ייעודי לצוות בלבד ב־`/#/settings` (לא מקושר מהממשק
  להורים): QR להדפסה כשילוט; ללא הורדה וללא הרשמה.
- **משוב הורים** — שאלון קצר ואנונימי.

שליחת מידע לצוות (אינטגרציה הורה↔צוות) לא מומשה בשלב זה, לפי האפיון —
הכרטיס מוצג פיזית על ידי ההורה בלבד.

## סטאק

- React 18 + TypeScript + Vite
- `react-router-dom` עם **HashRouter** (תואם GitHub Pages ללא הגדרות שרת)
- Client-side בלבד, ללא backend. אין איסוף מידע מזהה.
- כל נתוני המשתמש (סיבת הגעה, שלב במסע, כרטיס התאמות, משוב) נשמרים
  ב־`localStorage` עם תפוגה של 24 שעות — כך שרענון/חזרה באותו ביקור שומרים הכל,
  וביקור חדש מתחיל נקי.
- RTL מלא, גופן Assistant, טיפוגרפיה רספונסיבית (root font-size ב־clamp).

## הרצה מקומית

```bash
npm install
npm run dev        # שרת פיתוח — נפתח ב-http://localhost:5173/
npm run build      # בדיקת טיפוסים + build לפרודקשן אל dist/
npm run preview    # תצוגה מקדימה של ה-build (תחת /accessible-er/)
```

## פריסה ל־GitHub Pages

ב־build בלבד נקבע `base: '/accessible-er/'` ([vite.config.ts](vite.config.ts));
בפיתוח מקומי האפליקציה מוגשת ב־`/`.
קיים workflow ב-[.github/workflows/deploy.yml](.github/workflows/deploy.yml) שבונה
ומפרסם אוטומטית בכל דחיפה ל-`main` (יש להפעיל Pages → Source: GitHub Actions).

## מבנה

```
src/
  data/        # תוכן דמו: מסלולים, שלבים, פרוצדורות, מפה, תמיכה
  hooks/       # usePersistentState, useVisitReason
  components/   # AppShell, Mascot, רכיבי UI משותפים
  pages/       # דפי האפליקציה
  lib/         # עזרי localStorage עם תפוגה
```
